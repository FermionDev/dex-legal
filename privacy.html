DEX Privacy Policy

Effective Date: April 26, 2026

DEX is a local-first personal operating system developed and operated by FermionDev ("we", "us"). DEX runs entirely on the user's own device. This policy describes what data DEX handles and how.

1. Local-Only Architecture

DEX is desktop software installed and run on the user's machine. All user data — including financial ledgers, notes, calendar entries, contacts, and connected-service credentials — is stored locally on the user's device under the application's data directory. We do not operate servers that receive, store, or process user data. We have no access to user data unless the user explicitly transmits it to us (e.g., by emailing a support log).

2. Information Handled by DEX

  • Financial data via Plaid. When the user connects a bank account through Plaid Link, Plaid returns an access token and account/transaction data to DEX. The access token and transaction data are stored locally on the user's device in the application's data directory. DEX never receives or stores the user's bank login credentials — those are handled exclusively by Plaid.
  • Financial data via QuickBooks Online. If the user connects QuickBooks Online via OAuth, the OAuth tokens and synchronized accounting data are stored locally on the user's device.
  • Bank statement files. If the user imports OFX/QFX/CSV files, those files and the resulting ledger entries are stored locally.
  • User-generated content. Notes, calendar events, contacts, recipes, health logs, and other content created by the user are stored locally.
  • Optional cloud AI features. DEX is local-first by default. Cloud language model providers (e.g., Anthropic, OpenAI) are off by default and are only used when the user explicitly opts in to a specific feature and provides their own API key. When enabled, the relevant prompt content for that feature is sent to the provider the user selected, governed by that provider's privacy policy. The user can disable cloud AI at any time. DEX does not proxy these requests through our servers.

3. What We Do Not Collect

  • We do not collect analytics, telemetry, or usage metrics.
  • We do not operate user accounts, authentication servers, or cloud sync.
  • We do not sell, rent, or share user data with advertisers or data brokers.
  • We do not have access to the user's financial accounts, transactions, or credentials.

4. Third-Party Services Invoked by the User

When the user connects a third-party service, data flows directly between the user's device and that service. The user's relationship with each third party is governed by that third party's terms and privacy policy:

  • Plaid Inc. — bank connectivity. https://plaid.com/legal/
  • Intuit (QuickBooks Online). — accounting sync. https://www.intuit.com/privacy/statement/
  • Optional AI providers the user enables (e.g., Anthropic, OpenAI), each governed by their own policy.

5. Consent

All third-party connections are user-initiated. The user explicitly authenticates with each provider through that provider's official flow (Plaid Link, OAuth) before any data is exchanged. The user may revoke any connection at any time from within DEX or directly with the provider.

6. Data Retention and Deletion

Because data is stored locally, retention is controlled entirely by the user.

  • Disconnect a Plaid item: DEX calls Plaid's /item/remove endpoint to revoke the access token and removes the item from local state.
  • Wipe Plaid data: DEX provides a "Wipe" action that revokes every Plaid access token and removes all Plaid-sourced ledger entries from local storage.
  • Delete all data: The user may delete the application's data directory at any time, or uninstall DEX, which removes all locally stored data. See Delete Your Data for instructions.

7. Security

Data is stored on the user's own device and inherits that device's security posture (disk encryption, OS account controls, etc.). Network requests to third-party providers (Plaid, QuickBooks, AI providers) use HTTPS. Access tokens are stored in the local application data directory. Users are responsible for securing the device on which DEX is installed.

8. Children

DEX is not directed to children under 13 and we do not knowingly handle data from children.

9. Changes to This Policy

We may update this policy from time to time. The "Effective Date" above will reflect the latest revision. Material changes will be reflected in the application's release notes.
